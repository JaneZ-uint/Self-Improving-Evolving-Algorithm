Why It Works

为什么自我提升理论上可行？

生成 vs 判别：一种关键的能力不对称

生成（Generation）

从零写出正确答案

难

判别（Discrimination）

判断答案是否正确

相对容易

数学

写证明：需要创造性洞察
验证证明：逐步检查逻辑链

代码

写正确代码：要兼顾逻辑与边界
找 bug：运行测试用例即可

翻译

翻译出信达雅：需要深度理解
判断翻译好坏：对比即可感知

自我提升的本质：用更强的判别能力来指导更弱的生成能力。 当两者追平时，提升到达天花板。

Self-Instruct：自己给自己出题

Aligning Language Models with Self-Generated Instructions

"LLM 能不能像学生一样，自己给自己出练习题来做？"
Key Insight

你不需要从零开始。只需要 175 条种子任务（seed tasks） 作为"火种"， 模型就能自动生成成千上万条新的指令-响应对，然后用这些数据微调自己。

Result

从 175 条种子出发，最终生成了 52K 条 高质量指令数据， 微调后的模型在多个 benchmark 上接近 GPT-3.5。

质量控制与局限

质量控制策略

  • • ROUGE 去重，过滤相似指令
  • • 让 LLM 自己判断输出质量
  • • 过滤过于简单或无效的指令
  • • 混合种子任务与生成任务训练

局限性

  • • 无法提升推理能力
  • • 生成数据可能强化已有偏见
  • • 缺乏对"过程"的监督
  • • 模型犯的错会被写入训练数据
Zelikman et al., 2022

STaR：自举推理能力

Self Taught Reasoner: Bootstrapping Reasoning With Reasoning

"如果模型能解出这道题，那它的解题过程就是最好的训练数据。
如果解不出呢？——给它看答案，让它事后解释。"

就像考试不会做的题，看了答案之后觉得
"啊我本来也想到了" —— STaR 就是把这个过程形式化了。

STaR Dual-Path

双路径流程

答对了 → 直接用；答错了 → 看答案再解释

Innovation

Rationalization：即使模型答错了， 给它正确答案后让它"事后解释"，这个解释也能作为有效的训练数据。

STaR Example

数学推理示例

Problem

一个水池有两个进水管。A 管单独注满需要 6 小时，B 管单独注满需要 4 小时。如果两管同时打开，多久能注满？

1
Model's Wrong Attempt
6 + 4 = 10. 10 hours
Nope. The answer is 2.4 hours.
2
Rationalization (given answer = 2.4h)
A's rate = 1/6 pool/h, B's rate = 1/4 pool/h Combined rate = 1/6 + 1/4 = 5/12 pool/h Time = 1 / (5/12) = 12/5 = 2.4 hours
3
This rationale becomes training data

即使模型一开始答错了，rationalization 产生的推理过程也能帮助模型学会正确的工作记忆（chain-of-thought）。

Rationalization 为什么有效？

洞察 1：隐性知识

模型其实"知道"正确答案的路径，只是初始采样时没找到。 给它答案后，它能反向构建合理的推理链。

洞察 2：渐进提升

每一轮训练后，模型答对的题目越来越多， 需要 rationalization 的越来越少——形成正向循环。

迭代改进示例：代码质量

v1 · Initial
def flatten(lst): result = [] for item in lst: if isinstance(item, list): for sub in item: result.append(sub) else: result.append(item) return result
Critique: Only flattens one level. Fails on deeply nested input like [[1, [2, 3]], 4].
v2 · After 1 Refinement
def flatten(lst): result = [] for item in lst: if isinstance(item, list): result.extend(flatten(item)) else: result.append(item) return result
Critique: Recursive approach handles deep nesting. But ignores tuples — flatten([1, (2, 3)]) returns [1, (2, 3)].
v3 · Final
def flatten(lst): """Flatten a nested list/tuple structure.""" result = [] for item in lst: if isinstance(item, (list, tuple)): result.extend(flatten(item)) else: result.append(item) return result
Critique: Handles both list and tuple. Clean recursive logic with docstring. Acceptable.

Background

EM 算法：期望最大化

Expectation-Maximization Algorithm

一句话概括

给定含有隐变量的不完全数据， 反复迭代两个步骤来逼近参数的最优估计。

E-Step（期望步）

Expectation

  • • 固定当前参数 θ(t)
  • • 推断隐变量的期望分布
  • • "假设参数是对的，隐变量应该是什么？"

M-Step（最大化步）

Maximization

  • • 固定隐变量的期望分布
  • • 更新参数 θ(t+1) 使似然最大
  • • "给定隐变量的分布，参数该怎么调？"
直觉类比

假设你有两枚外观一样的硬币 A、B，混在一起抛了若干次，但不知道每次用的是哪枚。 E-step：根据当前估计的正面概率，推断每轮更可能用的是哪枚硬币； M-step：根据推断结果，重新估计两枚硬币的正面概率。 反复迭代，估计越来越准。

EM 算法保证似然单调递增——每轮迭代都至少不比上一轮差。

ReSTEM Perspective

EM 算法视角

E-Step

Expectation

  • • Grow：当前模型生成多个候选解
  • • Improve：用奖励模型评分筛选
  • • 保留高分样本，丢弃低分样本
  • • 相当于"期望"最好的输出分布

M-Step

Maximization

  • • Train：用筛选后的数据微调模型
  • • 模型参数更新，生成能力增强
  • • 下一轮 E-Step 能采样到更好的解
  • • 相当于"最大化"似然函数
Why EM Matters

EM 算法有单调递增的似然保证—— 每一轮迭代都不会让模型变差。这为 ReST 的有效性提供了理论支撑。

SPIN：自我博弈微调

Self-Play Fine-Tuning Converts Weak LMs to Strong LMs

AlphaGo 的启示

AlphaGo Zero 不依赖人类棋谱，完全通过自我对弈从零学到了超越人类的水平。

SPIN 问：语言模型能不能也和自己下棋？

核心思想

目标：让新模型的输出与人类数据不可区分

Self-Rewarding LM：自己当裁判

Self-Rewarding Language Models

"这就像你自己给自己的论文打分——听起来很不靠谱，但意外地 work。"
Core Breakthrough

不再需要单独训练的奖励模型（Reward Model）。LLM 自己既当选手（生成回答）又当裁判（评分评判）。 生成能力和评判能力在迭代中共同进化。

Self-Rewarding · 迭代提升

LLM-as-a-Judge + 迭代自进化

迭代流程

1 模型生成多个候选回答
2 LLM-as-Judge 自评打分
3 筛选高分回答作训练数据
4 微调模型 → 新一轮迭代
Evaluate the response on a scale of 1-10 based on: helpfulness, correctness, coherence...

RLHF vs Self-Rewarding

RLHF = 固定 RM → 天花板
Self-Rewarding = RM + Policy 共同进化 → 可能突破天花板
Bai et al., 2022 · Anthropic

Constitutional AI：给自我进化加上"宪法"

Harmlessness from AI Feedback

问题

RLHF 需要大量人类标注偏好数据，尤其对安全性的标注很难获取——你不能让人类去评价有害内容。

解法

用一组人类定义的原则（Constitution）代替大量人工标注。模型根据原则自我批评、自我修订。

Sycophancy Problem

模型倾向于"讨好"评判者而非追求真正的正确性。 当评判者就是自己时，这个问题更加严重——因为模型知道自己"喜欢"什么。

分布偏移：误差的滚雪球效应

Telephone Game

就像电话传话游戏——每传一次偏差一点。 第 1 个人说"下午三点开会"，到第 10 个人变成"下午上甜点开慧"。 自训练的每一轮都在"传话"，误差不断累积。

三个问题的恶性循环

以 AI 写作模型为例：三者同时发生、互相加强

Vicious Cycle

输出变窄 → 评分被套路 → 偏差放大 → 输出更窄 → ... 这就是为什么不加干预的自训练注定会退化。

Theory

一个 60 分的模型能把自己训练到 90 分吗？

可以，如果...

  • • 模型规模足够大
  • • 有可靠的验证信号（数学答案可验证）
  • • 生成中有足够的多样性
  • • 判别能力 > 生成能力（有提升空间）

— Huang et al., 2022

不行，如果...

  • • 缺乏外部验证信号（开放式生成任务）
  • • 生成多样性不足（sampling 太窄）
  • • 评判器本身有系统性偏差
  • • 生成能力 ≈ 判别能力（没有差距可利用）
Callback to Part A

还记得 Part A 讲的"批改作业比写作业容易"吗？ 自我提升的上界 = 判别能力追平生成能力的那一刻。 就像一个学生，当他的"审美"和"动手能力"一样强时，自学就到了瓶颈。

AI Safety

自我提升的安全隐忧

"如果一个 AI 能反复改进自己，
我们如何确保它改进的方向是人类想要的？"

风险

  • • Recursive Self-Improvement
    能力指数增长 → 人类来不及监管
  • • Goal Drift
    优化目标在迭代中悄悄偏移
  • • Value Lock-in
    错误的价值观被固化放大

对策

  • • Constitutional AI
    用原则/宪法约束自我进化方向
  • • Human-in-the-Loop
    关键节点保留人类审查权
  • • Interpretability
    让自我改进过程可解释可审计
